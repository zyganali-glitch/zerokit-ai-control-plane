# Codex session evidence for Devpost

The operator must run `/feedback` in the primary Codex task used for the final Build Week work. Codex opens the feedback flow and provides the Session ID required by the Devpost submission form.

- Paste that Session ID into the private Devpost field.
- **Do not commit the Session ID** to this public repository, README, issue, screenshot, or video.
- The Session ID is separate from a Codex subscription and separate from GPT-5.6 access.
- Its purpose is to help OpenAI and Devpost identify the Codex session used to build the submission.
- If multiple Codex tasks were used, choose the task where the majority of the submitted core functionality was built. For this repository, use the continuing primary project task that contains the final official-rules, Pages, product, test, and submission work—not a short throwaway task.
- The recording should show the bounded Codex task, visible GPT-5.6 selection, generated target, validator, and manifest. The Session ID itself does not need to appear publicly.

The repository cannot generate or validate this identifier. Keep it only in the private submission form or a private operator note.
